Topic Modeling (LDA & BERTopic)¶

Introduction¶

Customer feedback is an important asset for businesses. It can inform us what consumers like and dislike about our products and services. Because Webex is a keystone of Cisco's business model, it is imperative we understand how end-users feel about the product so we can better tailor it to their desires and needs.

To gauge what end-users think about Webex, there is a wealth of information stored in comments from post-meeting surveys that can be extracted. However, it is too tedious to parse through each verbatim comment manually. So, how do we do any of this? I have turned to data science and machine learning to automate the process of dissecting comments and grouping them into relevant topics. This is possible through topic modeling.

Background and Topics¶

Latent Dirichlet Allocation (LDA) topic modeling is a way to identify topics that best describe a set of documents (or comments, in this case). You can read more about it here. But instead of getting too into the weeds explaining its technical nature and how it works, I figured it would be best to give some background information and apply this knowledge to a case study.

The LDA model was trained using comments ($>$ 20 characters in length) from Webex App (WA) surveys between January 1st, 2021 and March 31st, 2022. The model determined that there were 9 topics (each consisting of 10 words) that best encapsulate what was being discussed in this collection of comments. Testing and Procedure¶

The validity of the model was tested using WA survey comments from April 1st, 2022 to June 22nd, 2022. The comments were processed using the following procedure...

  1. Tokenization (split into list of separate words)
  2. Removal of stop words (common words such as I, the, there, etc.)
  3. Lemmatization (retrieval of root words to simplify speech)
  4. Removal of unnecessary part-of-speech (POS) tags like punctuation, numbers, symbols, etc.

Then...

  • Sentiment was determined using the package Vader (i.e., was it positive, negative, neutral, etc.)
  • Comments were assigned one to three topics and those topics were ranked based on relevancy
  • Number of comments with a particular topic were counted
  • Visuals were created to show how topic distribution within the comments varied by different parameters such as NPS category, sentiment, and date

Leveraging BERTopic¶

Now that we have these nine topics from the LDA topic model, how can we extract more information from the comments?

BERTopic is a type of topic modeling that utilizes Transformers (architectures that encode and decode sentences), as well as other modeling techniques like UMAP and HDBSCAN, to create dense "clusters" of words which become easily-identifiable topics. You can read more about it here.

BERTopic is another method leveraged here (in conjuction with LDA topic modeling) to provide more robust and effective insight into the comments within each topic discussed above.

Code & Hyperparameters¶

# import packages for BERTopic
from umap import UMAP
from hdbscan import HDBSCAN
from bertopic import BERTopic
from sentence_transformers import SentenceTransformer
from sklearn.feature_extraction.text import CountVectorizer

# initialize sub-models for BERTopic
embedding_model = SentenceTransformer('all-MiniLM-L6-v2')
umap_model = UMAP(n_neighbors = 3, n_components = 3, min_dist = 0.01)
hdbscan_model = HDBSCAN(min_cluster_size = 50, min_samples = 5, gen_min_span_tree = True, prediction_data = True)

stop_words.add('don') # don't want to include this
stop_words.add('doesn') # same as above

vectorizer_model = CountVectorizer(ngram_range = (1, 2), stop_words = stop_words)

# create BERTopic model
model = BERTopic(umap_model = umap_model, 
                 hdbscan_model = hdbscan_model, 
                 embedding_model = embedding_model, 
                 vectorizer_model = vectorizer_model, 
                 top_n_words = 5, 
                 language = 'english', 
                 calculate_probabilities = True, 
                 verbose = True)

The DFs below highlight common words and/or bigrams (groups of two words) associated with each topic from BERTopic. Because of the natural invariability between each LDA topic and the comments within them, there is a different number of categories and word frequency determined by BERTopic for each topic derived from the LDA model.
